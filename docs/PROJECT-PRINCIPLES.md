# Project Principles

1. Review the approved documentation before major work.
2. Prepare a detailed plan and wait for explicit approval.
3. Create a rollback-safe backup before modification.
4. Implement one milestone at a time.
5. Preserve existing functionality and public identifiers where practical.
6. Test automatically and in a real WordPress staging environment.
7. Never commit secrets, production databases, local backups, or environment files.
8. Keep users in control of medium- and high-risk actions.
9. Extend the architecture through modules, registries, migrations, hooks, and documented contracts.
10. Update documentation and release evidence with every stable save point.
